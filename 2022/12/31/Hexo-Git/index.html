Git

在博客文件夹下右键鼠标，选择 Git Bash Here

  1. 创建草稿

    草稿文件夹 ...\Blog\source\_drafts 中的草稿并不会直接发布到网站上

  2. 发布已完成的草稿

    该命令会把 \source\_drafts 下的文章移到 \source\_posts 下

  3. 直接创建新文章

    创建的新文章将会保存在 ...\Blog\source\_posts 文件夹下

  4. 生成静态网站并部署

    clean 清除缓存文件 (db.json) 和已生成的静态文件 (public)。在某些情况（尤其是更换主题后），如果发现您对站点的更改无论如何也不生效，您可能需要运行该命令。

    generate 生成静态文件。

    deploy 部署网站。

  5. 查看

    默认情况下，访问网址为： http://localhost:4000/。
